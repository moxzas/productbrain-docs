The ProductBrain API lets LLM agents and scripts read, search, and modify your project. It's the same API that Claude Code, Cursor, and other AI agents use to operate on your planning tree.

<Note>
**Agents and LLMs: read the [LLM Guide](https://productbrain.com/docs/llm-guide.md) first.** It is the complete, canonical contract — every endpoint, field, validation rule, and worked example. This page is a human-friendly summary and **does not list the full surface**: view control, the hosted Workflow API, share-link lifecycle, tree navigation, and the in-band response-coaching layer all live in the guide. Give your agent that one URL and your API key and it has everything.
</Note>

## Base URL

```
https://productbrain.com/api/v1
```

## Authentication

All requests require a Bearer token — a per-user API key generated from the app.

```bash
curl -H "Authorization: Bearer pb_your_api_key" \
  https://productbrain.com/api/v1/nodes?projectId=your-project
```

Keys start with `pb_`, are scoped to your projects, and can be generated from **Settings → API Keys**. See [Authentication](/api-reference/authentication) for details.

## Endpoints at a Glance

### Search nodes by meaning

```bash
GET /api/v1/search?projectId=my-project&q=onboarding+flow&limit=3
# → { results: [{ node: { id, type, data }, similarity: 0.85, ancestors: [...] }] }
```

### List nodes (with filters)

```bash
GET /api/v1/nodes?projectId=my-project&type=job&iteration=MVP
# → { nodes: [{ id, type, parentId, data }], _meta: { currentIteration, nodeCounts } }
```

### Create a node

```bash
POST /api/v1/mutate
{ "action": "add", "projectId": "my-project",
  "node": { "type": "job", "parentId": "approach-1",
    "data": { "label": "Upload progress reaches 100%", "iteration": "MVP" } } }
# → { success: true, node: { id: "job-42", ... } }
```

### Update a node

```bash
POST /api/v1/mutate
{ "action": "update", "projectId": "my-project",
  "nodeId": "job-42", "data": { "status": "done" } }
```

### Get current iteration

```bash
GET /api/v1/iterations?projectId=my-project&current=true
# → { iteration: { name: "MVP", status: "active" } }
```

### Navigate the tree

```bash
GET /api/v1/tree?projectId=my-project&nodeId=approach-1&direction=context
# → { ancestors: [...], siblings: [...], children: [...] }
```

Full details for these: [Nodes](/api-reference/nodes) · [Search](/api-reference/search) · [Mutations](/api-reference/mutations) · [Iterations](/api-reference/iterations). For the rest of the surface — view control, the Workflow API, share-link lifecycle, and the `_meta` coaching layer — the [LLM Guide](https://productbrain.com/docs/llm-guide.md) is the complete reference.

## For LLM Agents

This API is designed as a **session contract for LLM agents**. Your agent reads the documentation, operates the tree, and the session ends. There are no client-side dependencies to maintain — no SDK to update, no webhooks to rewire, no version pinning. Changes to the API are reflected in the documentation immediately. Your agent picks them up on the next session.

The [LLM Operations Guide](https://productbrain.com/docs/llm-guide.md) is the canonical reference for any AI agent operating on ProductBrain. It covers the data model, hierarchy rules, field validation, placement workflow, and worked examples for generating a brain from a codebase or artifact.

Give your agent this URL and your API key — it has everything it needs to get started.

## Versioning & Stability

The API is versioned in the URL path — `https://productbrain.com/api/v1` is the current version. The commitment:

- **We will not break your integration without warning.** Breaking changes (removed or renamed fields, endpoints, or actions; tightened validation; changed error shapes) only ship in a new version (`/api/v2/...`).
- **The previous version keeps working for at least 6 months** after a new version ships.
- **Breaking changes are announced in the [changelog](/changelog/2026-06-11) at least 30 days** before the new version goes live.
- **Additive changes** (new endpoints, new optional fields, new actions) ship on the current version without notice — write your integration to tolerate unknown fields.

Older unversioned paths (`/api/nodes`, ...) keep working indefinitely and return a `Deprecation: true` header plus a `Link` header pointing at the `/api/v1` successor. If your client sees those headers, switch the base URL to `/api/v1` — nothing else changes.

This contract is enforced mechanically: a frozen contract test suite runs in CI against every release candidate, and a breaking change blocks the deploy.

## Your Data Is Yours

You can download your entire project as JSON at any time — every goal, need, approach, job, task, and phase. No export restrictions, no premium-only data access, no lock-in. The API that your agents use to read and write the tree is the same API you use to export everything.

```bash
# Download your full project
curl -s "https://productbrain.com/api/v1/nodes?projectId=my-project" \
  -H "Authorization: Bearer pb_..." > my-project-backup.json
```

This is a commitment, not a feature. We believe a product should compete on the value it creates, not the cost of leaving. If ProductBrain stops being useful, your data leaves with you in one command.
