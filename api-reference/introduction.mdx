The ProductBrain API lets LLM agents and scripts read, search, and modify your project. It's the same API that Claude Code, Cursor, and other AI agents use to operate on your planning tree.

## Base URL

```
https://productbrain.com/api
```

## Authentication

All requests require a Bearer token — a per-user API key generated from the app.

```bash
curl -H "Authorization: Bearer pb_your_api_key" \
  https://productbrain.com/api/nodes?projectId=your-project
```

See [Authentication](/api-reference/authentication) for details on key generation, scoping, and rate limits.

## Endpoints

| Endpoint | Method | Purpose |
|----------|--------|---------|
| [`/api/nodes`](/api-reference/nodes) | GET | List nodes (filterable by type, iteration) |
| [`/api/search`](/api-reference/search) | GET | Semantic search over nodes |
| [`/api/mutate`](/api-reference/mutations) | POST | Create, update, or delete nodes and iterations |
| [`/api/iterations`](/api-reference/iterations) | GET | List iterations (filterable by status) |
| `/api/tree` | GET | Navigate the tree — ancestors, siblings, children |
| `/api/projects` | GET | List your projects |

## Key Principles

<CardGroup cols={2}>
  <Card title="Search first" icon="magnifying-glass" href="/api-reference/search">
    Use `/api/search?q=...` to find nodes by meaning. Don't pull all nodes and filter — search uses vector embeddings and returns ranked results with ancestor paths.
  </Card>
  <Card title="One write endpoint" icon="bolt" href="/api-reference/mutations">
    All writes go through `/api/mutate`. The `action` field determines the operation: `add`, `update`, `delete`, or `batch`.
  </Card>
</CardGroup>

## For LLM Agents

The [LLM Operations Guide](https://productbrain.com/docs/llm-guide.md) is the canonical reference for any AI agent operating on ProductBrain. It covers the data model, hierarchy rules, field validation, placement workflow, and worked examples for generating a brain from a codebase or artifact.

Give your agent this URL and your API key — it has everything it needs to get started.
