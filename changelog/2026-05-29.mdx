## Highlights

### Onboarding wizard

New users no longer land on a blank canvas. A guided entry point offers two paths — **chat** (describe what you're working on) or **paste** (drop in existing text). The AI builds your first goal, needs, and approaches through conversation, then hands off to the main canvas.

### One AI engine behind every creation flow

All five creation flows now run through a single AI engine. The conversation is more natural: the AI classifies what you're saying, confirms its understanding, then generates a structured draft. Consistent quality across all entry points.

### Start from anywhere

The +Need button accepts whatever you throw at it — a solution, a problem, a goal, or something vague. The AI figures out what level you're at and maps it to the right place in the tree.

### Jobs are proof, not tasks

Jobs are now framed as **verifiable acceptance criteria** — observable proof that an approach is working. "Price comparison shows for scanned barcode" not "Add API call to fetch prices."

### Smarter API for agents

The API now guides agents toward the right endpoints. Wrong-door requests get a redirect. Large responses nudge toward semantic search.

## Other improvements

- Wizard waits for project data before rendering
- Draft nodes always visible in the focused view
- Viewport fits all visible nodes during AI conversations
- View state restored after commit/discard
- New jobs default to Later
- Smoother project switching
