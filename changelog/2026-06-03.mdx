## Highlights

### Learn the model by exploring it

New users now walk through an interactive example before the wizard. A coffee shop planning tree demonstrates how goals, needs, approaches, and jobs connect — click each node, see how they relate. Teaches the framework through a real example, not a wall of text.

### Easier ways to add work

The header now has **+Add** and **+Task**. +Add opens a chat that places nodes anywhere in the tree — describe what you're thinking and the AI figures out where it goes. +Task creates standalone items without changing what you're looking at.

### Create a single job from any approach

Select an approach, click +Job. Add one verifiable checkpoint — AI-assisted or type it yourself. No need to generate a full batch when you just want to add one thing.

### Simpler dimming

In iteration view, nodes are either full brightness or dimmed. One level. Dimmed means "not active in this iteration" — no need to interpret different shades.

## Other improvements

- Overview background refined — clearer distinction between modes
- Needs without jobs in the selected iteration start collapsed
- Task cards match the canvas background
- Later is the default — no separate "Unassigned" option
