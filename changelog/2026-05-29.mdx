## Highlights

### Onboarding wizard

New users no longer land on a blank canvas. A guided entry point offers two paths — **chat** (describe what you're working on) or **paste** (drop in existing text). The AI builds your first goal, needs, and approaches through conversation, then hands off to the main canvas.

### AI-assisted creation rebuilt from the ground up

All five creation flows — +Goal, +Need, inline Need, inline Approach, and the wizard — now run through a single AI engine. The conversation is more natural: the AI classifies what you're saying, confirms its understanding, then generates a structured draft. Quality is consistent regardless of which button you click.

### Start from anywhere

The +Need button accepts whatever you throw at it — a solution, a problem, a goal, or a vague idea. The AI figures out what level you're speaking at, maps it to the right place in the tree, and fills in the hierarchy above and below. You don't need to think in framework language.

### Jobs are proof, not tasks

Jobs are now framed as **verifiable acceptance criteria** — observable proof that an approach is working. "Price comparison shows for scanned barcode" instead of "Add API call to fetch prices." The AI generates proof-of-delivery language, and the UI reinforces this framing throughout.

### Smarter API for agents

The API now guides AI agents toward the right endpoints. Agents that try to search through the wrong endpoint get a clear redirect. Large responses include a nudge toward semantic search. Iteration queries explain how to find the current active iteration. This prevents agents from pulling entire trees when they should be using targeted lookups.

## Other improvements

- Wizard waits for your project to load — no flash of empty state
- Draft nodes always visible in the focused view
- Canvas viewport fits all visible nodes during AI conversations
- View state restored after committing or discarding drafts — you see what you just created
- New jobs and tasks default to the Later iteration
- Project switching is smoother — no page reload
